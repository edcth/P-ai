## 与原计划相比的实际落地差异

### 1. 权限卡最终不是简单复选框列表

- 最初目标只是“能勾选并保存”。
- 实际联调中，原生 checkbox 与 DaisyUI 选中态在当前结构下出现了明显的视觉不一致，导致“数据已写入但 UI 像没勾上”。
- 最终实现改为“整卡自定义选中态 + 左侧状态标记”，并把模式语义直接编码进视觉层。

### 2. 部门页最终补成了草稿保存模式

- 草案只要求“新增权限卡并生效”。
- 实际实现中，为避免编辑过程中直接污染全局配置，部门页同步改成显式保存的草稿模式。
- 结论：本轮不仅是权限卡新增，也顺带完成了部门页配置写回策略收口。

### 3. 固定头部被抽成公共容器

- 草案只提到部门页改造，没有要求整理配置页布局。
- 实际实现中，为对齐供应商页体验，抽出了 `SettingsStickyLayout.vue` 作为公共固定头部容器。
- 结论：本轮额外完成了一次配置页结构收口，而不是只在部门页里局部修补。

### 4. 系统部门动作语义被拆成两层

- 计划阶段原本只有“还原”一个动作概念。
- 实际联调中，顶部按钮承担“回退到当前草稿之前的状态”，详情卡按钮承担“恢复初始化默认值”，两者语义不同。
- 结论：最终文案已拆分，详情卡内系统部门动作固定为“恢复初始化”。
