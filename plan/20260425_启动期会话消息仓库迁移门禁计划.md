# 启动期会话消息仓库迁移门禁计划

## 1. 目标

现有会话仍落在：

```text
chat/conversations/<conversation_id>.json
```

新消息仓库目标是：

```text
chat/conversations/<conversation_id>/
  manifest.json
  meta.json
  messages.jsonl
  messages.idx.json
```

这次计划单独处理“启动期迁移门禁”，目标是：

- 启动时自动检查旧会话是否需要迁移。
- 迁移前先预验证。
- 预验证全部通过时自动迁移，并显示迁移进度。
- 预验证失败时前端明确报错，询问是否抛弃异常会话继续迁移。
- 迁移完成后才允许进入应用主界面。
- 正常运行期不再对“已进入迁移状态但失败/损坏”的会话静默降级。

## 2. 核心原则

- 迁移是启动门禁，不是业务运行中的懒加载副作用。
- 未进入迁移状态的 legacy JSON 只允许在迁移阶段读取。
- 只要会话目录存在 `manifest.json`，就必须以 manifest 状态为准。
- `ready JSONL` 才是正常运行期可读真相。
- `building / failed / rollback / stale / ready 但校验失败` 都必须阻断读取并给出明确错误。
- 用户选择“抛弃异常会话继续迁移”时，必须有清晰列表和日志，不静默删除。

## 3. 启动流程

启动流程调整为：

```text
Tauri 启动
  -> 初始化基础窗口/迁移界面
  -> 后端执行 message store preflight
  -> 如果无需迁移：进入应用
  -> 如果可自动迁移：显示迁移进度并执行迁移
  -> 如果存在异常：显示异常列表，等待用户选择
       -> 用户取消：退出或停留错误页
       -> 用户确认抛弃异常：备份/隔离异常会话，继续迁移其他会话
  -> 迁移完成：刷新缓存，进入应用
```

## 4. 后端接口

新增 Tauri command：

```rust
check_message_store_migration() -> MessageStoreMigrationPreflightReport
run_message_store_migration(input: RunMessageStoreMigrationInput) -> MessageStoreMigrationRunReport
```

建议结构：

```rust
struct MessageStoreMigrationPreflightReport {
    total_conversations: usize,
    ready_count: usize,
    legacy_count: usize,
    blocked_count: usize,
    can_auto_migrate: bool,
    items: Vec<MessageStoreMigrationPreflightItem>,
}

struct MessageStoreMigrationPreflightItem {
    conversation_id: String,
    title: String,
    status: String,
    message_count: usize,
    reason: Option<String>,
}

struct RunMessageStoreMigrationInput {
    discard_invalid: bool,
}

struct MessageStoreMigrationRunReport {
    migrated_count: usize,
    skipped_ready_count: usize,
    discarded_count: usize,
    failed_count: usize,
}
```

## 5. 进度事件

迁移运行时后端向前端 emit：

```text
message-store-migration-progress
```

建议 payload：

```rust
struct MessageStoreMigrationProgressPayload {
    current: usize,
    total: usize,
    conversation_id: String,
    title: String,
    status: String,
    detail: Option<String>,
}
```

状态建议：

```text
checking
migrating
skippedReady
discarded
failed
completed
```

## 6. 预验证规则

对每个会话：

### 6.1 Ready JSONL

如果目录型 shard manifest 为：

```text
storeKind = jsonlSnapshot
migrationState = ready
```

则校验：

- `meta.json` 可读。
- `messages.jsonl` 可读。
- `messages.idx.json` 可读。
- manifest 的消息数量、最后消息 id、jsonl 字节数与实际一致。

通过则标记为 `ready`。

失败则标记为 `blocked`，不能降级读旧 JSON。

### 6.2 Legacy JSON

如果没有 manifest，且存在 `<conversation_id>.json`：

预验证：

- JSON 可解析为 `Conversation`。
- conversation id 与文件名一致。
- conversation id 可作为目录型 shard 名。
- messages 可编码成 JSONL。
- 可生成 index。
- 目标目录可创建或不存在冲突。

通过则标记为 `legacyReadyToMigrate`。

失败则标记为 `blocked`。

### 6.3 非 ready manifest

如果 manifest 存在，但状态不是 ready：

```text
building / failed / rollback / none / unsupported
```

标记为 `blocked`。

不能自动回退 legacy。

## 7. 抛弃异常会话

用户确认“抛弃异常会话继续迁移”时，不能直接静默删除。

建议策略：

```text
chat/conversations/<id>.json
chat/conversations/<id>/
```

移动到：

```text
backups/discarded-message-store-migration/<timestamp>/<id>...
```

如果移动失败，再阻断迁移并显示错误。

必须记录日志：

```text
[消息存储迁移] 抛弃异常会话：conversation_id=...，backup=...，reason=...
```

## 8. 运行期读取规则

迁移门禁完成后，运行期规则改为：

```text
ready JSONL:
  正常读取。

没有 manifest 且存在 legacy JSON:
  理论上不应出现；返回错误提示“会话尚未迁移”，要求重新运行迁移。

有 manifest 但非 ready:
  返回错误，不降级。

ready 但校验失败:
  返回错误，不降级。
```

注意：开发阶段可以保留临时兼容开关，但默认行为必须是严格模式。

## 9. 前端启动体验

启动界面新增迁移状态页：

```text
检查会话数据...
正在迁移会话消息仓库 3 / 42
正在处理：xxx
```

预验证失败时显示：

- 失败会话数量。
- 每个失败项的标题、id、原因。
- 按钮：
  - 取消启动
  - 抛弃异常会话并继续迁移

迁移完成后自动进入应用。

## 10. 缓存处理

迁移完成后必须：

- 清理或刷新 `cached_app_data`。
- 清理或刷新 `cached_conversations`。
- 刷新 `cached_conversation_mtimes`。
- 刷新 `cached_app_data_signature`。
- 确保后续读取走 ready JSONL。

## 11. 实现阶段

### 阶段 1：后端 preflight

- 扫描 `chat/conversations`。
- 识别 legacy JSON、ready JSONL、blocked manifest。
- dry-run 验证 legacy 是否可迁移。
- 返回结构化报告。
- 增加单元测试。

### 阶段 2：后端 run migration

- 执行 legacy JSON -> JSONL snapshot。
- emit progress。
- 支持 `discard_invalid`。
- 迁移完成后刷新缓存。
- 增加迁移成功、失败、抛弃异常测试。

### 阶段 3：前端启动门禁

- 应用启动时先调用 preflight。
- 可自动迁移时显示进度并运行迁移。
- blocked 时显示确认弹窗/页面。
- 完成后再进入现有应用。

### 阶段 4：严格读取规则

- 运行期禁止 stale/failed/building manifest 静默降级。
- 运行期遇到 legacy JSON 返回“未迁移”错误。
- 保留测试或开发辅助入口用于构造 legacy 夹具。

## 12. 验证

最低验证：

- 全 legacy 且合法：启动自动迁移完成后进入应用。
- legacy JSON 损坏：前端显示异常项。
- 用户选择抛弃异常：异常文件进入 backup，其他会话迁移完成。
- ready JSONL 正常：跳过迁移。
- ready JSONL 损坏：启动阻断，不回退 legacy。
- building manifest：启动阻断。
- 迁移完成后 `cargo check` 通过，相关后端测试通过。

## 13. 与现有计划关系

本计划是 `20260424_消息JSONL外置与压缩段分页重构计划` 的前置门禁计划。

只有启动迁移门禁完成后，才能继续推进：

- 运行期严格读取。
- AppData 去消息正文化。
- message store 分页/索引读取优化。
