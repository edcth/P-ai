# 2026-04-20 系统提示词管理与部门缓存合并计划

## 说明

本计划用于合并并替代以下两份旧计划：

- `20260420_部门提示词内存缓存与按需重建计划.md`
- `20260420_系统提示词与对话提示词解耦及部门缓存计划.md`

当前代码已经不是只做“部门提示词缓存”，也不是停留在“讨论 system / conversation 分层”的阶段，而是已经实际落地了：

- system prompt 与 conversation prompt 的职责拆分
- `PromptManager` 风格的 system prompt 管理与缓存
- 部门块缓存
- 会话环境块缓存
- 系统提示词显式分层与顺序收口
- 工具规则按当前部门 / 当前 API / 当前可用工具条件化注入

因此需要一份按真实实现收口后的计划，作为后续测试、归档与回顾依据。

注意：

- 本文件当前仍放在 `plan/`，不进入 `plan/done/`
- 等待用户明确反馈“通过”后，再整理归档稿并移动到 `plan/done/`

## 背景

原始问题有两层：

1. system prompt 与 conversation prompt 在实现结构上长期混在一起
2. 部门相关提示词属于高频读取、低频变化内容，适合做缓存，但此前没有独立管理层

随着这轮实现推进，需求已经收口为：

- 保留既有对话提示词链路
- 为 system prompt 增加独立管理层
- 把部门相关块与工具规则块从 system prompt 中显式抽出
- 按当前部门权限与当前会话环境动态拼装 system prompt

## 合并后的目标

1. 明确 system prompt 与 conversation prompt 是两条不同构建链
2. 新增 `PromptManager` 风格的 system prompt 管理入口
3. 对部门相关 system prompt 块做内存缓存
4. 对会话环境相关 system prompt 块做独立内存缓存
4. 让 system prompt 的各组件顺序显式化、可测试
5. 将工具规则从“一坨说明”拆成按工具粒度、按可用性注入
6. 把 IM 规则、运行环境规则与工具规则解耦
7. 保证 system prompt 预览内容与真实运行拼装顺序一致

## 真实实现边界

### 一、conversation side

conversation / message side 继续负责：

- `history_messages`
- `latest_user_text`
- `latest_user_meta_text`
- `latest_user_extra_blocks`
- `latest_images`
- `latest_audios`

这些内容仍属于对话提示词，不属于 system prompt 本体，也不进入部门缓存。

### 二、system side

system prompt 现在收口为独立链路，核心由 `PromptManager` 统一组装。

system prompt 当前按语义拆成以下层次：

1. 固定系统准则
2. 部门提示词
3. 工具规则
4. 特殊工具运行环境说明
5. IM 规则

### 三、两级缓存边界

当前按两级重建粒度收口：

#### 1. 部门提示词缓存

缓存的是 system prompt 中稳定的部门相关块：

1. 部门提示词块
2. 只受部门权限影响的工具规则块

触发重建的主要条件：

1. 部门配置变化
2. 部门权限变化
3. 部门上下文变化

#### 2. 会话环境提示词缓存

缓存的是 system prompt 中与当前会话环境绑定的块：

1. shell / workspace 运行环境块
2. AGENTS.md 注入相关块
3. IM 规则块

第一版环境变更检查采用轻量签名，不做重扫描。签名来源包括：

1. `conversation_id / conversation_kind`
2. 当前 shell 运行时信息
3. 当前会话工作目录与会话工作区配置
4. `AGENTS.md` 的存在性、文件大小、修改时间
5. 已传入的运行环境块与 IM 运行块摘要

当前不进入这两级缓存的内容：

1. 固定系统准则
2. 用户消息、历史消息、多模态消息
3. conversation side 的 latest user / history / attachments

## 已完成实现

### 1. PromptManager 入口已建立

已新增：

- `src-tauri/src/features/chat/prompt_manager.rs`

负责：

- system prompt 组件顺序装配
- 部门块缓存
- system prompt 缓存
- 最终 preamble 构建

### 2. system / conversation 已实际拆开

当前实现里：

- system side 由 `finalize_system_prompt_with_manager(...)` 统一完成
- conversation side 继续保留在 `PreparedPrompt` 其余字段

这意味着后续讨论“提示词缓存”时，已经可以明确区分：

- system prompt
- conversation prompt

### 3. 部门与会话环境缓存已接入

当前已存在三级缓存结构：

- `DepartmentSystemPromptSnapshot`
- `ConversationEnvironmentPromptSnapshot`
- `SystemPromptCacheEntry`

已接入：

- 部门块缓存命中
- 会话环境块缓存命中
- system prompt 缓存命中
- 相关中文日志

### 4. 工具规则已拆分为按工具注入

原先一大段 `system tools rule` 已拆分。

当前 system prompt 会按：

- 当前部门权限
- 当前系统工具默认开关

决定是否注入：

- memory 相关规则
- `plan` 工具规则
- `meme` 规则
- 其他 builtin tool 规则块

这一步已经实现“不是所有工具规则都无脑注入”。

### 5. question / planning 已从固定系统块移出

`<question and planning rule>` 已经不再属于固定系统准则，
而是按语义归入 `plan` 工具规则，并且只有工具可用时才注入。

### 6. IM 规则与运行环境规则已分开

当前已做的纠偏包括：

- 普通会话不再无条件注入 `remote im contact rules`
- 只有远程联系人会话才会出现联系人语义规则
- `remote im runtime activation` 继续作为 IM 运行态块单独存在

### 7. 运行环境块已按当前配置 shell 输出

此前 `shell workspace` 块错误使用启动时探测 shell。

现已修正为按当前配置的运行时 shell 输出，避免：

- 明明配置为 PowerShell
- 预览里却显示 Git Bash

### 8. system prompt 标签与格式已统一

本轮已统一多项核心 XML 标签与显示格式，包括：

- 核心标签英文命名
- system prompt 块间不再插入空行
- 预览结构更稳定、更便于检查
