## 背景

当前终端权限判断依赖字符串扫描与路径猜测，已出现对正常读取命令的误拦截。典型问题包括：

- 将重定向 token 误识别为普通路径
- 将错误输出重定向误判为写入意图
- 路径提取、写入风险判断、工作目录权限校验职责交叉

目标是重构为“先分析命令语义，再做路径与权限判断”的统一链路，达到更稳定、可测试、可扩展的效果。

## 目标

1. 新增统一的终端命令语义分析器
2. 将 Bash / Git Bash 与 PowerShell 的路径、重定向、目录切换、读写意图分析结构化
3. 让工作目录权限校验只消费分析结果，不再自行猜路径
4. 保留现有审批体系，但提升触发准确率
5. 为分析器补充充分测试，覆盖常见读写、重定向、sink、目录切换与未知场景

## 范围

### 包含

## 设计原则

1. 命令分析与权限判断分层
2. 普通路径参数、重定向目标、sink 分开处理
3. 对明确可分析的命令给出精确结论
4. 对无法可靠分析的命令走保守路径，但不误伤明显安全的读取命令
5. 测试优先，分析器输出应易于断言

## 计划实现

### 第一部分：统一分析结果模型

新增统一 IR，至少表达：

- 读取路径
- 写入路径
- 新建路径
- 删除路径
- 重定向目标
- sink 目标
- 目录切换
- 未知风险标记

### 第二部分：Bash / Git Bash 分析

覆盖：

- 普通读取命令，如 `ls`、`cat`、`find`、`rg`
- 写入命令，如 `cp`、`mv`、`rm`、`touch`、`mkdir`
- 输出重定向
- `2>/dev/null`、`>/dev/null` 等 sink
- `&&`、`||`、`;`、`|` 组合
- `cd && ...` 目录切换场景

### 第三部分：PowerShell 分析

覆盖：

- Cmdlet 参数表驱动的路径识别
- 写入类 cmdlet
- 读取类 cmdlet
- 重定向目标
- `Set-Location` 等目录切换
- Windows sink 场景

### 第四部分：接入现有权限判断

改造 `exec` 现有链路：

- 路径提取改为消费分析器结果
- 写入风险判断改为消费分析器结果
- 工作目录匹配只对结构化路径集合执行
- sink 不再进入路径越界与写入风险判断

### 第五部分：测试

新增并覆盖以下测试组：

1. 读取命令
2. 写入命令
3. 输出重定向
4. `/dev/null` / `nul` sink
5. 相对路径与绝对路径
6. `cd` / `Set-Location` 目录切换
7. PowerShell 参数路径识别
8. 未知命令与保守降级
9. 现有致命误拦截场景回归

## 完成标准

1. 正常读取命令不会再因 sink 或重定向被误判为越界/写入
2. 写入命令只对真实写入目标做权限校验
3. 分析器模块具备充分单元测试
4. 现有 `exec` 权限链路完成接入
5. 代码结构比当前更清晰，后续规则扩展不再依赖字符串猜测
